# Single Layer Perceptron - Banknote Authentication Classification

CS3807 ‚Äì Deep Learning Laboratory, Experiment 1 - Single Layer Perceptron

A Single Layer Perceptron built from scratch in NumPy to classify banknotes as authentic or forged, using the Banknote Authentication dataset, followed by a comparison against scikit-learn's Perceptron.

## Overview

This project covers the full pipeline end to end:

- Dataset exploration (first samples, dataset shape, missing values, descriptive statistics, class distribution)
- Exploratory data analysis (histograms, correlation heatmap, pairwise scatter plots, boxplots)
- Preprocessing (feature scaling with `StandardScaler`, stratified 80/20 train/test split)
- A Single Layer Perceptron built from scratch - weight and bias initialization, step activation, forward propagation, and the Rosenblatt learning rule, with full epoch-by-epoch training logs
- Evaluation (accuracy, precision, recall, F1-score, confusion matrix)
- Step vs Sigmoid activation comparison, with a discussion of why Step cannot be used in backpropagation
- Learning rate comparison (0.001, 0.01, 0.1) using random weight initialization
- A two-feature decision boundary plot (optional task)
- A comparison against scikit-learn's `Perceptron`
- A demonstration that a single layer perceptron cannot solve the XOR problem
- A study of the effect of feature normalization on convergence speed

## Results

| Metric | From-Scratch Perceptron | Scikit-learn Perceptron |
|---|---|---|
| Accuracy | 98.55% | 97.45% |
| Precision | 96.83% | 94.57% |
| Recall | 100.00% | 100.00% |
| F1-score | 98.39% | 97.21% |

Model trained for 50 epochs at learning rate 0.01, on an 80/20 stratified split (1097 train, 275 test images). Both implementations reach perfect recall, and the from-scratch model slightly outperforms scikit-learn's Perceptron on accuracy, precision and F1-score. Of the three learning rates tried (0.001, 0.01, 0.1), 0.001 gave the best test accuracy at 98.91%.

## Dataset

Banknote Authentication Dataset ‚Äî UCI Machine Learning Repository, downloaded manually and placed alongside the notebook (no programmatic fetch, since the source repository is not reachable from every environment).

- 1372 rows total, 4 numeric features (variance, skewness, curtosis, entropy), 2 classes (0 = authentic, 1 = forged)
- Fairly balanced (762 authentic, 610 forged)
- No missing values

Download link: https://archive.ics.uci.edu/dataset/267/banknote+authentication

## Getting Started

### 1. Clone the repository

```
git clone https://github.com/<your-username>/<your-repo-name>.git
cd <your-repo-name>
```

### 2. Set up a virtual environment (recommended)

```
python3 -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
```

### 3. Install dependencies

```
pip install -r requirements.txt
```

### 4. Add the dataset

Download `data_banknote_authentication.txt` from the link above and place it in the project root, next to the notebook. It should have no header row, columns in the order variance, skewness, curtosis, entropy, class.

### 5. Run the notebook

```
jupyter notebook perceptron_lab.ipynb
```

or open it directly in Google Colab and run all cells top to bottom. No GPU is needed, the whole notebook runs in well under a minute on CPU.

## Building the Report

The LaTeX report lives in `Experiment_1_Report/Experiment_1_Report.tex` and pulls in the figures from `Experiment_1_Report/figures/`. The figures are PDF (not EPS), so no `-shell-escape` flag is required.

1. Upload `Experiment_1_Report/Experiment_1_Report.tex` and everything in `Experiment_1_Report/figures/` into an Overleaf project (same folder level - keep the `figures/` subfolder as is).
2. Compile with pdfLaTeX.

Or locally:

```
cd Experiment_1_Report
pdflatex Experiment_1_Report.tex
```

Text in the figures is set in Liberation Serif, a free substitute for Times New Roman that matches its letter widths and shapes almost exactly (Times New Roman itself is a Microsoft font and is not installed by default on Linux). If Times New Roman is available on your machine, change the `FONT_NAME` variable near the top of the notebook and re-run it to regenerate the figures.
