# Multi-Layer Perceptron - Fashion-MNIST Classification

CS3807 – Deep Learning Laboratory, Experiment 2 - Multi-Layer Perceptron

A Multi-Layer Perceptron built with TensorFlow/Keras to classify clothing images into 10 categories, using the Fashion-MNIST dataset, followed by automated hyperparameter optimization.

## Overview

This project covers the full pipeline end to end:

- Dataset exploration (sample images, class distribution)
- Preprocessing (flattening 28×28 images to 784-dim vectors, normalization to [0,1], one-hot encoding)
- A baseline MLP built in Keras - Dense(128, ReLU) → Dense(64, ReLU) → Dense(10, Softmax), with full epoch-by-epoch training logs
- Evaluation (accuracy, precision, recall, F1-score, confusion matrix, classification report)
- Automated hyperparameter optimization - random sampling of hyperparameter combinations (`ParameterSampler`) over hidden layers, hidden neurons, learning rate, batch size, epochs, optimizer, activation function, and dropout rate, each evaluated with 5-fold stratified cross-validation
- Retraining and evaluating the optimized model on the full training set
- A baseline vs. optimized comparison, with a second confusion matrix for the optimized model

## Results

| Metric | Baseline Model | Optimized Model |
|---|---|---|
| Accuracy | 87.53% | 88.10% |
| Precision | 87.65% | 88.13% |
| Recall | 87.53% | 88.10% |
| F1-score | 87.39% | 87.98% |
| Training Time | 166.43 s | 88.70 s |

Baseline trained for 20 epochs at batch size 32 (Adam, lr = 0.001). Optimized model uses the best configuration found by the search - 1 hidden layer, 256 neurons, Tanh activation, 0.2 dropout, Adam (lr = 0.001), batch size 128, 20 epochs (5-fold CV accuracy 84.22%) - retrained on the full 60,000-image training set and evaluated on the 10,000-image test set. The optimized model matches or exceeds the baseline on every metric while training in roughly half the time.

## Dataset

Fashion-MNIST Dataset — Zalando Research, fetched programmatically via `tensorflow.keras.datasets.fashion_mnist`, so no manual download is required.

- 70,000 grayscale images total (60,000 train / 10,000 test), 28×28 pixels, 10 clothing classes
- Classes perfectly balanced (6,000 training images per class)
- No missing values

## Getting Started

### 1. Clone the repository

```
git clone https://github.com/<your-username>/<your-repo-name>.git
cd <your-repo-name>
```

### 2. Set up a virtual environment (recommended)

```
python3 -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
```

### 3. Install dependencies

```
pip install -r requirements.txt
```

### 4. Run the notebook

```
jupyter notebook deep_learning_lab_2.ipynb
```

or open it directly in Google Colab and run all cells top to bottom. The notebook fetches the dataset itself - nothing needs to be downloaded by hand. No GPU is required, but training is faster with one.

## Building the Report

The LaTeX report lives in `report/main.tex` and pulls in the figures from `report/figures/`. The figures are PDF (not EPS), so no `-shell-escape` flag is required.

1. Upload `report/main.tex` and everything in `report/figures/` into an Overleaf project (same folder level - keep the `figures/` subfolder as is).
2. Compile with pdfLaTeX.

Or locally:

```

pdflatex main.tex
```
